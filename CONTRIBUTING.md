# Contributing to K-Guard

Thank you for your interest in improving K-Guard! To maintain high standards (Cisco/DevNet style), please follow these guidelines:

## 📝 Coding Standards
- **Language:** All code comments and documentation MUST be in **English**.
- **Python:** Follow PEP 8 style guides for the FastAPI backend.
- **Security:** Do not commit secrets. Use environment variables.
- **Pull Requests:** All changes must be submitted via a Pull Request and pass CI/CD scans (Trivy) before merging.

## 🚀 Development Process
1. Fork the repository.
2. Create a feature branch.
3. Commit your changes with descriptive messages.
4. Push to your branch and open a PR.
